任务描述"Kimi 配额预算：余 14-15 次"与实际不符。首次有效调用（已修编码）立即返回 429。推测其他并行代理（P4 几社 3 人、P4 画派 3 人、松江扩容代理等）已消耗完本 period 的配额，或配额单位并非"次"而是 token 量。

**建议**：
- 后续调度 Kimi 批任务前先做一次轻量 ping 调用（如 `kimi -p "输出: ok"`）探测配额状态
- `memory/project_kimi_timeout_issue.md` 可补充"配额耗尽 429 探测机制"

### 问题 2：Kimi CLI 在 Windows 下需 UTF-8 环境变量
默认 GBK 控制台下，Kimi 输出含 `•` `◄` `►` 等 Unicode 符号时会崩溃（rich 库的 `_win32_console.py` 无 fallback）。

**修复方案（已验证）**：调用前设置
```bash
export PYTHONIOENCODING=utf-8
export PYTHONUTF8=1
```

**建议**：
- 将此环境变量固化到 `tools/` 下的 Kimi 调用包装脚本，或者写入 hook（`update-config`）在 Kimi 相关命令前自动注入
- 这是一条工具能力边界的新增观测，应回流到 `CLAUDE.md` 的"工具能力边界（实战验证）—— Kimi"小节
