# 人物传记验收工具集

这是 `人物名录/` 下传记文件的自动化验收工具集。每次新增或修订一份
传记后，都应运行本目录的工具做结构、争议下沉、验收规则三层检查。

所有工具均为**只读**，不会修改 `.md` 传记文件，只写入 `_validation/`
目录下的 Markdown 报告和 JSON 结果。

## 目录结构

```
_validation/tools/
├── README.md                     # 本文件
├── run_validation.py             # 主入口，一键跑全部或指定规则
├── bio001_structure_check.py     # BIO-001 结构重排检测
├── bio003_dispute_scan.py        # BIO-003 争议下沉扫描
└── bio006_audit.py               # BIO-006 全库验收审计
```

## 工具说明

### `run_validation.py` —— 主入口

把下面三个规则串成一条流水线，可以一次跑全部，也可以只跑其中一个。
运行结束会打印各规则的通过率和退出码。

| 参数 | 说明 |
|------|------|
| `--base-dir <path>` | 人物名录根目录，**必需** |
| `--all` | 运行 BIO-001 + BIO-003 + BIO-006（默认） |
| `--bio001` | 只运行 BIO-001 结构检查 |
| `--bio003` | 只运行 BIO-003 争议下沉扫描 |
| `--bio006` | 只运行 BIO-006 全库验收审计 |
| `--output <path>` | 单规则模式下指定 Markdown 输出路径 |
| `--json-dir <dir>` | 把 BIO-001 / BIO-006 的 JSON 结果写入该目录 |
| `--exclude-file <file>` | BIO-003 的已修订文件清单 |
| `--quiet` | 仅输出统计 |

**常用命令**：

```bash
# 全量跑（报告自动落在 _validation/ 下）
python _validation/tools/run_validation.py --base-dir 人物名录 --all

# 只跑结构检查
python _validation/tools/run_validation.py --base-dir 人物名录 --bio001

# 只跑 BIO-006 并把报告落到自定义路径
python _validation/tools/run_validation.py \
    --base-dir 人物名录 --bio006 --output 验收报告.md

# 跑全部并把 JSON 集中到一个目录
python _validation/tools/run_validation.py \
    --base-dir 人物名录 --all --json-dir _validation/json
```

**退出码**：
- `0`：所有启用的规则均通过；
- `1`：任一规则不通过；
- `2`：参数或文件系统错误。

### `bio001_structure_check.py` —— BIO-001 结构重排检测

**规则**：传记应包含主干四段 + 尾部章节：

    基本信息 → 生平概要 → 关键事迹 → 争议与待考 → {来源汇总, 关联主题}

- **必备章节**（5 个）：基本信息、生平概要、关键事迹、争议与待考、来源汇总
- **可选章节**（1 个）：关联主题（缺失仅产生 warning，不导致不通过）
- **顺序要求**：主干四段严格按上面顺序出现；尾部的"来源汇总"和
  "关联主题"允许任意顺序，但必须都排在主干之后。
- **"基本信息"必须是全文第一个 `##` 二级章节。**

**支持的章节别名**（宽松匹配，子串命中即可）：

| 规范名 | 别名 |
|--------|------|
| 基本信息 | 基本信息 |
| 生平概要 | 生平概要 / 生平简介 / 生平 / 概述 |
| 关键事迹 | 关键事迹 / 主要事迹 / 关键事实 |
| 争议与待考 | 争议与待考 / 争议与存疑 / 存疑 & 待查 / 存疑&待查 / 存疑与待查 / 未决争议 / 待查 / 存疑 |
| 来源汇总 | 来源汇总 / 参考文献 / 主要来源 / 来源追踪 / 主要参考 / 史料来源 / 资料来源 / 引用来源 / 参考资料 |
| 关联主题 | 关联主题 / 相关主题 / 关联条目 / 相关条目 |

**输出**：`_validation/BIO-001结构检查报告.md`，含：
- 汇总统计（通过/不通过/warning）
- 不通过清单（缺失章节、顺序错误等具体错误）
- Warning 清单（仅缺可选章节）
- 全文件章节概览表

**命令**：

```bash
python _validation/tools/bio001_structure_check.py \
    --base-dir 人物名录 \
    [--output 报告路径.md] \
    [--json json路径.json] \
    [--quiet]
```

### `bio003_dispute_scan.py` —— BIO-003 争议下沉扫描

**规则**：凡出现 "存疑 / 待考 / 原始出处待查 / 来源：存疑 / 待核 /
野史 / 民间传说" 等关键词的行，必须位于"存疑 & 待查"章节（或其别名）
之内，不得散落在其他段落。

**过滤策略**：
- 存疑章节内的命中不算违规；
- 标题行本身不算命中；
- 元数据噪声不算违规，例如：
  - `本档案综合…`、`本报告综合…`、`本文档综合…`
  - `编制说明`、`编纂时间`
  - `— 野史 —`、`野史性质`、`民间传说与地方记忆`（来源分级标签）
  - `[YYYY-MM-DD 规范化]` 规范化注释
  - `已移入`、`已迁入`、`[原…存疑`、`(续…`（迁移提示）

**输出**：`_validation/BIO-003扫描报告.md`，逐文件列出违规行号、所在
章节和行内容。

**命令**：

```bash
python _validation/tools/bio003_dispute_scan.py \
    --base-dir 人物名录 \
    [--output 报告路径.md] \
    [--exclude-file 已修订清单.txt] \
    [--quiet]
```

**已修订清单文件格式**（`--exclude-file`）：

```
# 井号开头的行会被忽略
张居正.md
海瑞.md
李时珍.md
```

### `bio006_audit.py` —— BIO-006 全库验收审计

**规则**（六条）：

| 规则 | 说明 | 判定标准 |
|------|------|----------|
| R1 一手依据 | 每篇至少 2 条一手或高等级二手依据 | 从 `[来源：xxx]` 标注中识别高等级来源，或正文中 `明史/明实录/大明会典/墓志/行状/学者专著` 等关键词命中 ≥ 2 |
| R2 存疑搬运 | 正文无 `[来源：存疑]`、`(原始出处待查)` 类句内标注 | 存疑章节外不得出现这类标记 |
| R3 未决争议 | 文件末尾有"存疑 & 待查"章节且含实质条目 | 至少 1 条列表项或长度超过 10 的正文行 |
| R4 来源汇总 | 有独立的"来源汇总"章节且 ≥ 5 条 | 支持列表和表格两种写法 |
| R5 基本信息 | 生卒/字号/籍贯/身份 至少 3 项 | 在"基本信息"章节中通过正则识别 |
| R6 文件长度 | ≥ 100 行合格，100–200 行为"合格偏短" | < 100 行视为 fail |

**分类体系**：

- **全合规**：6 条规则全过且长度 ≥ 200 行
- **合格偏短**：6 条规则全过，长度 100–200 行（可接受，建议扩写）
- **部分不合规**：1–2 条规则失败，且不涉及 R1/R2
- **严重不合规**：R1 或 R2 失败，或 ≥ 3 条规则失败

**输出**：
- `_validation/BIO-006验收报告.md`（人类可读）
- `_validation/BIO-006审计结果.json`（机器可读，便于后续聚合）

**命令**：

```bash
python _validation/tools/bio006_audit.py \
    --base-dir 人物名录 \
    [--output 报告路径.md] \
    [--json json路径.json] \
    [--quiet]
```

## 新传记工作流集成

每次新增或修订一份传记后，建议的最小验收流程：

```bash
# 1. 全量跑一次
python _validation/tools/run_validation.py --base-dir 人物名录 --all

# 2. 查看报告
# _validation/BIO-001结构检查报告.md
# _validation/BIO-003扫描报告.md
# _validation/BIO-006验收报告.md

# 3. 修订完后再跑一次，退出码应为 0
python _validation/tools/run_validation.py --base-dir 人物名录 --all
```

详细的验收流程和通过标准，见
[`人物名录/_validation/新传记验收流程.md`](../新传记验收流程.md)。

以及 `tools/` 目录下的单独脚本。

## 环境要求

- Python 3.8+
- 标准库即可（不依赖任何第三方包）
- 在 Windows 控制台输出中文时，建议使用 `python -X utf8` 或设置
  `PYTHONIOENCODING=utf-8`，否则 stdout 可能出现乱码（不影响报告文件本身）。
