# 证据链规范（Evidence Chain Specification）

> 版本：v1.0
> 制定日期：2026-04-10
> 适用范围：`人物名录/` 下全部人物传记文件（含后续新增）
> 制定动机：BIO-002 校验暴露 50+ 条目引用百度/维基等二手来源、60+ 条目将存疑写作定论、同一事件跨文件出处不一致。来源分级警示已做到"事后标注"，但缺一套**前置的、结构化的**证据管理机制。

---

## 1. 设计原则

### 1.1 第一性原理

人物传记知识库的价值取决于两件事：**事实的准确性** 与 **出处的可追溯性**。前者决定知识能否被信任，后者决定错误能否被定位和修正。缺一不可。

因此每条"关键事迹"都必须满足两条硬约束：

- **可追溯**：读者能根据标注定位到原始文献的具体段落（卷次/册次/页码/链接）
- **可分级**：读者能一眼判断这条信息是来自一手正史还是二手百科

### 1.2 分级而非二分

过往的 BIO-002 校验把来源粗暴分为"高/中/低风险"，粒度太粗。实际史料可信度是连续谱：
- 《明史》和《万历野获编》同为古籍，但前者是官修正史、后者是私家笔记，可信度不同
- 同为现代二手，严肃学术专著和百度百科的可信度差两个数量级

因此需要**多级分级**，让标注本身承载可信度信息。

### 1.3 短期并存，长期统一

现有 67 份文件已有大量 `[来源：xxx]` 非结构化标注。一次性改全不现实。因此规范允许两种格式在短期内共存：
- **旧格式**：`[来源：《明史·张居正传》卷213]`
- **新格式**：`[证据：E1/《明史·张居正传》卷213/wikisource-link]`

BIO-006 审计脚本的新版本（BIO-007/008/009）将同时识别两种格式，对仍为旧格式的条目记为"待迁移"而非"不合规"。

## 2. 证据分级体系（E1–E6）

### 2.1 六级定义

| 等级 | 类别 | 典型来源 | 可信度 | 说明 |
|------|------|---------|--------|------|
| **E1** | 一手正史 | 《明史》、《明实录》（太祖—熹宗/毅宗各朝）、《国榷》、《明史纪事本末》、《明通鉴》 | 最高 | 官修或准官修的系统性正史，是明代史最核心的叙事来源 |
| **E2** | 实录档案 | 《万历起居注》、《筹海图编》、《纪效新书》、《练兵实纪》、《万历会计录》、《大明会典》、《大明律》、《皇明经世文编》、《国朝献征录》、官方奏疏文集 | 极高 | 成文于明代、直接记录朝政或具体制度/事件的原始档案 |
| **E3** | 碑志方志 | 墓志铭、神道碑、行状、行实、家谱、家乘、府志、州志、县志、碑刻拓本 | 高 | 带有地方性/家族性偏向但时间近、细节具体，对官修正史是重要补充 |
| **E4** | 学术论文 | 有明确作者、期刊、年份、出版社的现代研究（含专著、学位论文、期刊论文、会议论文、译著） | 中高 | 例：黄仁宇《万历十五年》、樊树志《晚明史》、韦庆远《张居正和明代中后期政局》、《明清史研究》期刊论文 |
| **E5** | 明清笔记 | 《万历野获编》、《酌中志》、《明季北略》、《谷山笔麈》、《五杂俎》、《菽园杂记》、《客座赘语》、《罪惟录》、《名山藏》 | 中 | 私家撰述，细节丰富但夹杂传闻，需与正史互校 |
| **E6** | 百科二手 | 百度百科、维基百科（中/英）、搜狐、知乎、自媒体、无作者标注的网页 | 低 | **最弱**。只能作为线索指引，不能作为事实来源单独使用 |

### 2.2 分级判定规则

面对一条来源时按如下顺序判定：

1. **是否为明代成文、官修或准官修的通史/正史？** → E1
2. **是否为明代成文、记录具体事件或制度的原始档案/法典/兵书/奏疏集？** → E2
3. **是否为明代或清代近世留存的墓志/方志/家谱/碑刻？** → E3
4. **是否为有明确作者、期刊/出版社、年份的现代学术研究？** → E4
5. **是否为明清两代的私家笔记/野史/稗史？** → E5
6. **其他（百科、自媒体、匿名网页、无从定位的网络转引）？** → E6

### 2.3 边界情况

- **《明儒学案》**：归入 E4（黄宗羲撰，但本质是学术性的思想史整理，而非一手档案）。如引用其中转录的明人原文，则按被转录的文本分级。
- **《剑桥中国明代史》**：E4（现代学术专著）。
- **《万历十五年》**：E4（现代学术专著）。
- **《四库全书总目提要》**：E4（官方目录学整理，属学术评述）。引用其收录的古籍原文时按原书分级。
- **《古今图书集成》**：E4（清代官修类书，引用其中条目时按被引用的原始文献分级）。
- **文物出版社、中华书局点校本古籍**：按原书分级（点校本不降级）。
- **维基文库（wikisource）上的古籍原文**：按原书分级，但 URL 作为可追溯链接附在证据标注末尾。
- **出处不明的"据说/相传/民间传说"**：E6，且**必须**加"待核"。

## 3. 证据标注格式

### 3.1 基本格式

```
[证据：等级/来源/定位]
```

三段之间用 `/` 分隔：
- **等级**：E1–E6 之一
- **来源**：规范的文献名称（古籍须含卷次，现代著作须含作者）
- **定位**：可追溯的额外信息（链接、页码、藏本、出版社版本等），可留空但不省略斜杠

### 3.2 示例

**E1 一手正史**：
```
[证据：E1/《明史·张居正传》卷213/]
[证据：E1/《明史·海瑞传》卷226/中华书局点校本1974年版]
[证据：E1/《明神宗实录》卷125/https://zh.wikisource.org/wiki/...]
```

**E2 实录档案**：
```
[证据：E2/《大明会典》卷二十/万历重修本]
[证据：E2/《皇明经世文编》卷324·张江陵集·陈六事疏/]
[证据：E2/《筹海图编》卷十一/嘉靖刻本]
```

**E3 碑志方志**：
```
[证据：E3/张居正墓志铭/上海博物馆藏拓本]
[证据：E3/《江陵县志·人物志》/光绪刻本]
[证据：E3/《琼州府志》卷32/清道光重修本]
```

**E4 学术论文/专著**：
```
[证据：E4/韦庆远《张居正和明代中后期政局》/广东高等教育出版社1999年版p.287]
[证据：E4/黄仁宇《万历十五年》/生活·读书·新知三联书店1997年版第一章]
[证据：E4/陈义钟《海瑞年谱》/中华书局1962年版]
```

**E5 明清笔记**：
```
[证据：E5/《万历野获编》卷8/中华书局1959年版]
[证据：E5/《谷山笔麈》卷4/]
[证据：E5/《酌中志》卷14/]
```

**E6 百科二手**：
```
[证据：E6/百度百科"张居正"条目/待核]
[证据：E6/维基百科中文"李时珍"条目/待核]
```

### 3.3 多来源并列

当一条事迹有多个来源时，用分号分隔，按等级从高到低排列：

```
[证据：E1/《明史·海瑞传》卷226/；E4/陈义钟《海瑞年谱》/中华书局1962年版]
```

### 3.4 一句话条目的标注位置

标注紧跟在事实陈述末尾，中间用空格分隔：

```
- 万历六年（1578）下令清丈全国田亩，颁布《清丈条例》。 [证据：E1/《明史·食货志》卷77/]
```

## 4. 硬性规则

### 4.1 关键事迹章节

| 规则 | 内容 |
|------|------|
| **R1** | 每条"关键事迹"至少需要 1 条 E1/E2/E3 级证据 |
| **R2** | 只有 E6 级证据支撑的事迹，必须在证据标注末尾加"待核"，且在文末"存疑 & 待查"节单独列出 |
| **R3** | E4 级证据可独立支撑事迹，但同一事迹如存在可用的 E1/E2/E3，应优先引用 E1/E2/E3 |
| **R4** | E5 级证据仅可作为**辅助**来源，不能单独支撑关键事迹；必须至少搭配一条 E1/E2/E3/E4 |
| **R5** | 任何条目同时使用多个等级证据时，证据链标注必须按等级从高到低排列（便于读者快速判断可信度） |

### 4.2 存疑与争议章节

| 规则 | 内容 |
|------|------|
| **R6** | "存疑 & 待查"节的条目**不受 R1 约束**——因为这些条目本身就是因为证据不足才被列入 |
| **R7** | "争议与待考"节的每一条必须包含"各方主要证据"子段落，列出至少两派的代表性证据（包括其等级） |

### 4.3 基本信息与人际关系

| 规则 | 内容 |
|------|------|
| **R8** | "基本信息"节（生卒/字号/籍贯/官职）如有争议，争议项须加证据标注；无争议项可以省略 |
| **R9** | "重要人际关系"节的每一条人际关系若用于后续事迹推导，须按关键事迹标准标注证据 |

## 5. 迁移策略

### 5.1 不要求一次性改全

现有 67 份文件均含 `[来源：xxx]` 非结构化标注。一次性重写成本过高，且容易引入新错误。本规范采用**渐进式迁移**：

- **新写文件**：必须从一开始使用 E1–E6 证据标注
- **修订已有文件**：在例行修订（校验报告发现问题、用户要求增补）时顺便迁移
- **主动迁移**：按 §5.2 的优先级排序，逐份迁移

### 5.2 优先级排序

按下列顺序主动迁移：

**优先级 1（高风险文件，立即迁移）**：
- BIO-002 标注"高风险"的 28 份文件
- BIO-006 审计中 R1（一手依据）不合规的文件
- 因百科来源被读者质疑过的文件

**优先级 2（重要人物，半年内迁移）**：
- 阁臣重臣全部（14 份）
- 帝王全部
- 关键武将与文人学者（按历史影响力）

**优先级 3（次要人物，一年内迁移）**：
- 其余全部文件
- 新增文件

### 5.3 迁移步骤（单份文件）

1. **识别**：扫描文件内所有 `[来源：xxx]` 标注
2. **分级**：按 §2.2 判定规则将每个来源映射到 E1–E6
3. **补足**：E1/E2/E3 来源尝试补充卷次/册次/链接
4. **降级标注**：E6 来源加"待核"标记
5. **并存**：无法当场分级的来源保留旧格式 `[来源：xxx]`，审计脚本会识别为"待迁移"
6. **更新时间戳**：在文件顶部加 `> [2026-XX-XX 证据链迁移]：已迁移 N 条证据，剩余 M 条待处理`

### 5.4 自动化迁移脚本（正则雏形）

以下为可用的正则匹配规则，用于**自动预分级**：人类仍需最终审核，但能把 80% 显而易见的情况处理掉。

```python
# evidence_migrate.py —— 正则雏形，非完整可运行脚本
import re

# E1：一手正史
E1_PATTERNS = [
    r"《明史[·・]?[^》]*》(?:卷\d+)?",
    r"《明[世穆神熹武宪光毅]宗实录》(?:卷\d+)?",
    r"《明实录》",
    r"《国榷》(?:卷\d+)?",
    r"《明史纪事本末》(?:卷\d+)?",
    r"《明通鉴》(?:卷\d+)?",
]

# E2：实录档案/法典/兵书
E2_PATTERNS = [
    r"《万历起居注》",
    r"《筹海图编》(?:卷\d+)?",
    r"《纪效新书》",
    r"《练兵实纪》",
    r"《大明会典》(?:卷\d+)?",
    r"《大明律》",
    r"《万历会计录》",
    r"《皇明经世文编》(?:卷\d+)?",
    r"《国朝献征录》(?:卷\d+)?",
    r"《万历邸钞》",
    r"《[^》]*奏疏》",
    r"《[^》]*全集》",  # 官员文集（需二次确认）
]

# E3：碑志方志
E3_PATTERNS = [
    r"[^》]*墓志(?:铭)?",
    r"[^》]*神道碑",
    r"[^》]*行状",
    r"[^》]*行实",
    r"《[^》]*(?:府|州|县|乡)志》",
    r"《[^》]*通志》",
    r"[^》]*家谱",
    r"[^》]*家乘",
]

# E4：现代学术
E4_AUTHORS = [
    "黄仁宇", "樊树志", "卜正民", "牟复礼", "傅衣凌",
    "韦庆远", "陈宝良", "陈义钟", "孟森", "吴晗",
]
E4_PATTERNS = [
    r"《剑桥中国明代史》",
    r"《万历十五年》",
    r"《明代[^》]*制度》",
    r"《明史研究[^》]*》",
]

# E5：明清笔记
E5_PATTERNS = [
    r"《万历野获编》(?:卷\d+)?",
    r"《酌中志》(?:卷\d+)?",
    r"《明季北略》",
    r"《明季南略》",
    r"《谷山笔麈》(?:卷\d+)?",
    r"《五杂俎》",
    r"《菽园杂记》",
    r"《客座赘语》",
    r"《罪惟录》",
    r"《名山藏》",
    r"《弇州史料》",
]

# E6：百科二手
E6_PATTERNS = [
    r"百度百科",
    r"维基百科",
    r"搜狐",
    r"知乎",
    r"自媒体",
    r"^[^《]{0,20}$",  # 无书名号、长度极短的疑似百科
]


def classify(source_text: str) -> str:
    """返回 E1–E6 之一。未命中则返回 'UNKNOWN'。"""
    for p in E1_PATTERNS:
        if re.search(p, source_text):
            return "E1"
    for p in E2_PATTERNS:
        if re.search(p, source_text):
            return "E2"
    for p in E3_PATTERNS:
        if re.search(p, source_text):
            return "E3"
    for author in E4_AUTHORS:
        if author in source_text:
            return "E4"
    for p in E4_PATTERNS:
        if re.search(p, source_text):
            return "E4"
    for p in E5_PATTERNS:
        if re.search(p, source_text):
            return "E5"
    for p in E6_PATTERNS:
        if re.search(p, source_text):
            return "E6"
    return "UNKNOWN"


def migrate_line(line: str) -> str:
    """把 [来源：xxx] 替换为 [证据：Eх/xxx/]。"""
    def repl(m):
        src = m.group(1).strip()
        if "存疑" in src or "待查" in src or src == "":
            return m.group(0)  # 保留原样
        level = classify(src)
        if level == "UNKNOWN":
            return m.group(0)  # 保留原样，人工审核
        suffix = "/待核" if level == "E6" else "/"
        return f"[证据：{level}/{src}{suffix}]"
    return re.sub(r"\[来源[：:]([^\]]+)\]", repl, line)
```

**使用流程**：
1. `python evidence_migrate.py <file>` → 生成 `<file>.migrated` 预览
2. 人工审核 diff，确认分级无误
3. 覆盖原文件并提交

**脚本局限**：
- 无法处理多来源并列的情况（`[来源：A；B]` 需拆分）
- 无法自动补齐卷次（若原文缺失）
- `UNKNOWN` 条目须完全人工处理

## 6. 验收规则（BIO-007 / BIO-008 / BIO-009）

以下三条新规则将在 BIO-006 审计脚本的下一版本中实现，与既有 6 条规则并列。

### 6.1 BIO-007：证据分级覆盖

> **规则**：每份传记的"关键事迹"章节至少 80% 的条目有 E1–E4 级证据。

**判定方法**：
1. 提取"关键事迹"章节（及其子节，如"嘉靖朝""隆庆朝""万历朝"）下的所有列表项
2. 对每个列表项检测是否包含 `[证据：E1/` 到 `[证据：E4/` 的标注
3. 统计命中率 = 含 E1–E4 证据的条目数 / 总条目数
4. 命中率 ≥ 80% → 通过

**过渡期规定（2026 年内）**：
- 旧格式 `[来源：xxx]` 中命中 §5.4 的 E1–E4 正则的，同样计入命中
- 给每份文件出具"已迁移比率"指标，但不因未迁移而判不合规

### 6.2 BIO-008：证据可追溯

> **规则**：每条 E1–E3 级证据必须有具体卷次/册次/链接之一。

**判定方法**：
1. 扫描全文所有 `[证据：E1/...]`、`[证据：E2/...]`、`[证据：E3/...]` 标注
2. 对每一条，检测来源段中是否含下列任一：
   - `卷\d+`
   - `册\d+`
   - `第\d+卷`
   - `第\d+册`
   - `https?://`
   - `藏本|拓本|刻本|写本|抄本`
   - `p\.?\s*\d+` 或 `第\d+页`
3. 全部命中 → 通过；有一条未命中 → 列出问题条目

### 6.3 BIO-009：E6 降级

> **规则**：所有 E6 级证据必须同时有"待核"标注。

**判定方法**：
1. 扫描全文所有 `[证据：E6/...]` 标注
2. 对每一条，检测是否在同一标注内包含"待核"字样
