## 8. 遇到的工程问题与沉淀

这些问题在本批未阻塞流程，但是在前序批次付出调试成本得到的硬规则，明确记入本报告以约束 P2/P3 续作。

### 8.1 问题 D：`run_in_background` + subprocess 的 2 分钟静默死亡

- **现象**：Claude Code 的 `run_in_background` shell 对长时间无 stdout 输出的进程有静默回收（约 2 分钟）。子孙进程被 Windows Job Object 一起 kill，kimi 调用被吞
- **规避**：本批所有 Kimi/Codex 调用都避开 `run_in_background + python 包 subprocess` 的组合
- **详细诊断**：见 `docs/biography_validation_pipeline_design.md` 第 16 节

### 8.2 Codex heredoc + 反斜杠续行卡死

- **现象**：Codex CLI 在部分 shell 下用 heredoc 结合反斜杠续行会卡在 stdin 读取上
- **规避**：本批全部使用"单行位置参数 + `< /dev/null`"方式调 Codex，未触发该问题
- **作为 P2/P3 调度规范**：Codex 调用一律采用单行位置参数，禁止 heredoc + 反斜杠续行组合

### 8.3 Kimi 日配额上限

- **实测**：约 17–18 次 / 日，超过触发 `rate_limit_reached_error`
- **规划原则**：单日批次人数 × 阶段数 ≤ 17，或拆分到跨日执行
- **本批验证**：6 人 × 1 次 Kimi 初稿 = 6 次，加上其他零散调用用尽当日配额，7 人顺延
